## 5. 安全设计

### 5.1 认证机制

```
┌─────────────────────────────────────────────────────────────────┐
│                      JWT 认证流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 登录                                                        │
│     Client ──POST /auth/login──> Server                         │
│                                  ├─ 验证凭证                     │
│                                  ├─ 生成 AccessToken (2h)        │
│                                  └─ 生成 RefreshToken (7d)       │
│     Client <────Tokens───────── Server                          │
│                                                                 │
│  2. 请求受保护资源                                               │
│     Client ──Header: Bearer <token>──> Server                   │
│                                        ├─ 验证Token签名          │
│                                        ├─ 验证Token有效期        │
│                                        └─ 解析用户ID             │
│     Client <────────Data───────────── Server                    │
│                                                                 │
│  3. Token刷新                                                   │
│     Client ──POST /auth/refresh──> Server                       │
│                                    ├─ 验证RefreshToken           │
│                                    └─ 生成新Token对              │
│     Client <────New Tokens─────── Server                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 安全措施

| 层级 | 措施 | 说明 |
|------|------|------|
| 传输层 | HTTPS | 全站SSL加密 |
| 认证层 | JWT | 双Token机制 (Access + Refresh) |
| 接口层 | Rate Limit | 限流防刷 |
| 数据层 | 参数化查询 | 防SQL注入 |
| 密码 | bcrypt | 密码哈希存储 |

```yaml
# docker-compose.yml
version: '3.8'
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - backend
      
  backend:
    image: studyflow-backend:latest
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_URL=jdbc:postgresql://postgres:5432/studyflow
      - REDIS_URL=redis://redis:6379
    depends_on:
      - postgres
      - redis
      
  postgres:
    image: postgres:16-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=studyflow
      - POSTGRES_USER=studyflow
      - POSTGRES_PASSWORD=${DB_PASSWORD}
      
  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:
```

## 7. 性能优化策略

### 7.1 前端优化

| 策略 | 实现方式 |
|------|----------|
| 代码分割 | React.lazy + Suspense |
| 资源缓存 | Service Worker + Cache API |
| 数据缓存 | TanStack Query stale-while-revalidate |
| 虚拟列表 | react-window (长列表) |

### 7.2 后端优化

| 策略 | 实现方式 |
|------|----------|
| 数据库索引 | 高频查询字段索引 |
| 查询优化 | 预聚合统计表 |
| 缓存 | Redis 热点数据缓存 |
| 连接池 | HikariCP 数据库连接池 |

### 7.3 缓存策略

| Key Pattern | 数据 | TTL | 失效时机 |
|-------------|------|-----|----------|
| `stats:today:{userId}` | 今日统计 | 60s | 番茄钟完成 |
| `stats:weekly:{userId}` | 周统计 | 5min | 每天更新 |
| `user:{userId}` | 用户信息 | 30min | 信息更新 |
| `tasks:today:{userId}` | 今日任务 | 不缓存 | 实时查询 |
