# StudyFlow 社区模块 PRD（双端）

> 版本: 1.0.0
> 日期: 2026-03-17
> 状态: 设计评审稿（开发前）
> 适用端: Web + Mobile

---

## 1. 背景与目标

StudyFlow 已具备任务、番茄钟、统计和个人中心能力，社区模块目标是把“个人学习”升级为“同伴激励学习”，提升留存与学习连续性。

**业务目标（MVP）**
- 提供学习动态发布与浏览能力，形成内容流。
- 提供点赞、评论等低门槛互动，提升活跃度。
- 提供学习小组入口，承接垂直兴趣学习人群。
- 与现有番茄钟/任务数据联动，支持“学习打卡”表达。

**成功指标（首月）**
- 社区 DAU 渗透率 ≥ 35%
- 动态发布率（日活用户发帖占比）≥ 8%
- 人均互动（点赞+评论）≥ 2.0 次/日
- 加入小组转化率 ≥ 15%

## 2. 用户与场景

**核心用户**
- 考研/考公用户：需要同伴监督、阶段打卡。
- 编程学习用户：重视方法分享、问题讨论。
- 语言学习用户：偏好日更打卡、轻互动。

**核心场景**
1. 我完成学习后，希望一键发布打卡并获得反馈。
2. 我浏览他人学习经验，点赞评论获取动力。
3. 我按目标加入学习小组，获取同频伙伴。

## 3. 范围定义

### 3.1 MVP（P0，首期）
- 动态流：列表、详情、分页、下拉刷新。
- 发动态：文本、标签、最多 9 张图、学习时长/任务数快照。
- 互动：点赞/取消点赞、一级评论、二级回复、删除本人评论。
- 小组：列表、详情（基础信息）、加入/退出。
- 个人社区页：我的发布、我的点赞（可后置为 P0.5）。

### 3.2 增强（P1）
- 话题聚合页、热门榜单。
- 学习排行榜（按周专注时长）。
- 小组管理（创建、审核、管理员权限）。
- 内容举报与审核后台。

### 3.3 非目标（本期不做）
- 私信系统
- 实时聊天室
- 长视频内容分发

## 4. 信息架构与交互主流程

**导航层级**
- 一级导航：社区
- 二级 Tab：动态广场 / 学习小组
- 关键页面：发帖页、帖子详情页、小组详情页

**关键流程**
1. 发布流程：进入发帖 -> 编辑内容与标签 -> 选择图片 -> 发布成功 -> 回流动态流顶部。
2. 互动流程：浏览动态 -> 点赞/评论 -> 实时更新互动计数。
3. 小组流程：浏览小组 -> 查看目标与成员数 -> 加入 -> 动态流按小组筛选。

## 5. 功能规则（核心）

### 5.1 发帖规则
- 内容长度 1~2000 字，前后空白自动裁剪。
- 图片支持 jpg/png/webp，单图 ≤ 5MB，最多 9 张。
- 标签最多 5 个，每个标签 ≤ 20 字。
- 学习快照字段由后端校验（防篡改，优先读取服务端统计）。

### 5.2 互动规则
- 点赞为幂等操作，重复请求不重复计数。
- 评论支持两级结构（comment + reply），后续按需扩展多级。
- 删除帖子/评论采用软删除，前端展示“内容已删除”占位。

### 5.3 小组规则
- 组员关系唯一（group_id + user_id）。
- 默认公开组可直接加入，私有组预留审核状态（P1）。
- 退组后保留历史发言与历史帖子归属。

## 6. 验收标准（DoD）

- Web/Mobile 均完成动态流、发帖、点赞评论、小组加入主链路。
- 接口响应时间：P95 < 500ms（不含图片上传）。
- 所有写接口具备鉴权、参数校验、权限校验。
- 关键计数字段（like_count/comment_count/member_count）与明细一致。
- 埋点可统计曝光、发布、点赞、评论、入组行为。

## 7. 依赖与风险

**依赖**
- 认证模块：复用 JWT 鉴权。
- 用户模块：复用用户资料（昵称、头像）。
- 统计模块：提供学习时长/任务快照数据。
- 文件存储：图片上传到 OSS/MinIO。

**风险与缓解**
- 热帖高并发点赞导致计数竞争：采用事务 + 原子更新。
- 不良内容风险：先做关键词过滤 + 举报入口预留。
- 双端体验不一致：统一共享类型与交互规范。

## 8. 里程碑建议

- M1（1 周）：数据模型与接口联调、动态流与小组列表打通。
- M2（1 周）：发帖/评论/点赞全链路完成，双端自测。
- M3（0.5 周）：性能优化、埋点补齐、灰度上线。
