## 已发现的问题及修正

### 1. API端点路径不一致 ✅ 已修复

**问题**：`packages/shared/src/constants/index.ts` 中番茄钟API路径使用单数形式 `/pomodoro/`，而 `docs/API_SPEC.md` 中使用复数形式 `/pomodoros/`

**修正**：统一修改为复数形式 `/api/v1/pomodoros/*`，符合RESTful规范

**文件变更**：
- `packages/shared/src/constants/index.ts`

### 2. 后端指南中数据库设计建议不一致 ✅ 已修复

**问题**：`docs/BACKEND_GUIDE.md` 建议将用户设置拆分到 `user_settings` 表，但 `docs/DATABASE.md` 和前端实际代码使用的是扁平化设计（所有字段在 `users` 表中）

**修正**：更新 `BACKEND_GUIDE.md`，明确推荐使用扁平化设计，并解释原因

**文件变更**：
- `docs/BACKEND_GUIDE.md`

### 3. API文档缺少重要说明 ✅ 已修复

**问题**：`docs/API_SPEC.md` 中没有强调用户资料接口的数据一致性要求

**修正**：在 3.12 节新增"用户资料接口重要说明"，详细说明：
- `GET /api/v1/users/profile` 必须返回完整数据
- 设置更新后必须同步更新 `updated_at` 字段
- 数据库采用扁平化设计的原因

**文件变更**：
- `docs/API_SPEC.md`

## 后续建议

### 1. 后端开发顺序建议

按照以下顺序进行开发，可以确保依赖关系清晰：

1. **基础设施层**
   - 数据库创建（PostgreSQL）
   - Redis 配置
   - Flyway 迁移脚本

2. **核心实体和基础配置**
   - Entity 类（User, Task, PomodoroRecord）
   - Repository 接口
   - 统一响应封装（ApiResponse）
   - 全局异常处理

3. **安全认证**
   - JWT 工具类
   - Spring Security 配置
   - AuthController（注册/登录/刷新Token）

4. **业务模块（按依赖顺序）**
   - User模块（用户资料、设置）
   - Task模块（任务管理）
   - Pomodoro模块（番茄钟）
   - Stats模块（统计计算）
   - Dashboard模块（聚合接口）

### 2. 测试策略

| 层级 | 测试类型 | 工具 | 覆盖率目标 |
|------|----------|------|------------|
| 单元测试 | Service/工具类 | JUnit 5 + Mockito | 80% |
| 集成测试 | Repository/API | TestContainers | 关键流程 |
| 契约测试 | API接口 | Spring Cloud Contract | 核心接口 |

### 3. 性能优化清单

- [ ] 数据库索引创建（已提供）
- [ ] Redis 缓存配置（用户资料、今日统计）
- [ ] 连接池调优（HikariCP）
- [ ] API 响应压缩
- [ ] 分页查询优化

## 快速开始

### 使用IDEA AI助手生成代码

1. 打开 IDEA，安装 AI 助手插件（如 GitHub Copilot、通义灵码）
2. 打开 `docs/BACKEND_ARCHITECTURE_GUIDE.md`
3. 复制第12节"AI 助手提示词模板"的内容
4. 在 AI 助手对话框中粘贴提示词
5. 根据生成的代码逐步完善项目

### 手动创建项目

1. 使用 [Spring Initializr](https://start.spring.io/) 创建项目
2. 选择依赖：Spring Web, Spring Data JPA, PostgreSQL, Redis, Spring Security, Flyway, Lombok
3. 按照 `BACKEND_ARCHITECTURE_GUIDE.md` 中的代码逐步实现
